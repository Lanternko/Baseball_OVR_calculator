# 棒球 OVR 計算器 📊⚾

> 一個基於真實 MLB 數據的棒球選手能力值評估和賽季表現模擬系統

## 🚀 快速上手

### 使用方式
本系統提供兩大核心功能頁面：

1. **📈 OVR 計算頁面**：輸入選手的打擊三圍（POW/HIT/EYE），系統會計算出綜合能力值(OVR)
2. **🎮 賽季模擬頁面**：基於三圍數值，透過蒙地卡羅模擬預測選手的完整賽季表現

### 🌐 線上體驗
直接開啟 GitHub Pages 連結即可立即使用，無需安裝任何軟體：
```
https://lanternko.github.io/Baseball_OVR_calculator/
```

### 🎨 設計風格
- **極簡日式美學**：乾淨簡潔的介面設計
- **響應式佈局**：完美支援桌面端和行動裝置
- **翻轉式球員卡**：模擬實體球員卡的翻轉動畫效果
- **專業數據呈現**：清晰易懂的統計數據展示

## 🎯 系統設計目標

### 目標 1：真實 MLB 表現範圍 (OVR 40-99)
- **98% 的選手** 應落在 OVR 40-99 區間內
- 涵蓋完整的 MLB 選手表現光譜

### 目標 2：幻想極值支援 (OVR 1-500)
- **OVR 1-40**：業餘水準（小聯盟、大學、業餘選手）
- **OVR 100-150**：傳奇水準，超越當前 MLB 標準
- **OVR 150-500**：純幻想/遊戲娛樂用途
- **OVR 500**：數學上限（.999 打擊率，3.999 長打率）
- 必須保持**平滑、遞增曲線**貫穿整個範圍

## ⚡ 屬性影響階層

### 💪 POW (長打力) 屬性
**主要影響 → 次要影響 → 第三影響**
1. **長打率 (SLG)** - *主要影響*
2. **長打數 (XBH)** - *高度影響*  
3. **全壘打數 (HR)** - *高度影響*
4. **二壘安打數 (2B)** - *中度影響* (在中等 POW 達到峰值，頂級時減少)
5. **打擊率 (AVG)** - *輕微影響* (直接影響很小)

**POW 效果**：
- 將一壘安打轉換為二壘安打（場邊長打力）
- 將二壘安打轉換為全壘打（頂級長打力）
- 輕微將高飛出局轉為二壘安打（不可防守的球）
- **注意**：需要良好的 HIT 技術作為基礎 - POW 放大現有接觸能力

### 🎯 HIT (接觸/技術) 屬性  
1. **打擊率 (AVG)** - *主要影響*
2. **場內安打率 (BABIP)** - *主要影響*
3. **三振率 (K%)** - *高度影響* (負相關)
4. **長打數 (XBH)** - *中度影響* (使 POW 發揮效果)
5. **長打率 (SLG)** - *中度影響* (長打力基礎)

**HIT 效果**：
- 核心接觸能力和球棒控制技巧
- 使 POW 能夠發揮效果（沒有接觸就沒有長打力）
- 透過更好的球棒控制減少三振

### 👁️ EYE (選球眼) 屬性
1. **保送率 (BB%)** - *主要影響*
2. **上壘率 (OBP)** - *主要影響*
3. **三振率 (K%)** - *高度影響* (負相關)
4. **全壘打數 (HR)** - *輕微影響* (透過耐心等待獲得正面修正)
5. **打擊率 (AVG)** - *輕微影響* (透過更好的球數間接影響)

**EYE 效果**：

- 球種識別和好球帶判斷
- 耐心等待有利球數
- 減少對壞球的追打率

## 📊 百分位點數系統

### 核心原理
- **POW/HIT/EYE 40** = 第1百分位 (PR1) - MLB 最低 1%
- **POW/HIT/EYE 70** = 第50百分位 (PR50) - MLB 平均水準  
- **POW/HIT/EYE 99** = 第99百分位 (PR99) - MLB 頂尖 1%
- **POW/HIT/EYE 100+** = 超越當前 MLB 標準

### 微調範例
- 選手打擊率 .185 → 調整 HIT 至 38 (低於 PR1)
- 選手打擊率 .222→ 調整 HIT 至 53 (高於 PR1)  
- 大尺度準確性比細微變化更重要

## ✅ 模型驗證要求

### 準確性目標
1. **POW 百分位在所有 HIT/EYE 組合中保持一致**
2. **基本功不足限制長打力效果** (HIT40/EYE40 上限)
3. **頂級基本功放大長打力效果** (HIT99/EYE99 乘數效應) 
4. **真實 MLB 分佈** 符合 2024 年合格選手數據
5. **幻想數值平滑遞增** 直到 OVR 500

### 關鍵驗證點
- **最低 1% 長打力**: ~15 支全壘打，.320-.350 長打率範圍  
- **平均長打力**: ~25 支全壘打，.420-.470 長打率範圍
- **頂級長打力**: ~40 支全壘打，.580-.650 長打率範圍
- **MVP 長打力**: ~50 支全壘打，.650-.700 長打率範圍
- **GOAT 長打力**: ~60-70 支全壘打，.750-.850 長打率範圍

## 🛠️ 實作注意事項

### 關鍵平衡點
- **OVR 40-99**: 必須代表 98% 的現實棒球表現
- **OVR 100-140**: 傳奇領域，但理論上仍可達成
- **OVR 150+**: 純幻想，數學級數遞增至完美
- **所有曲線**: 必須平滑、非遞減、內部一致

### 現實世界錨點  
- 使用 2024 年 MLB 合格選手 (420+ 打席) 作為現實基準
- Bonds 2001年 (.328/.515/.863) 作為 GOAT 級上限
- 數學極值 (.999/3.999) 作為 OVR 500 目標

## 🎨 技術特色

### 核心演算法
- **蒙地卡羅模擬**：透過大量隨機模擬產生統計數據
- **S 曲線插值**：平滑的非線性屬性效果轉換
- **機率分佈模型**：基於真實 MLB 事件機率

### 數據來源
- **2024 年 MLB 官方數據**：420+ 打席合格選手統計
- **歷史偉人數據**：Bonds、Ruth、Williams 等傳奇賽季
- **現代球星基準**：Judge、Ohtani、Soto 等當代頂尖選手

### 開發原則
- **現實主義優先**：OVR 40-99 必須反映真實棒球
- **平滑擴展性**：支援幻想數值而不破壞現實感
- **數學嚴謹性**：所有曲線保證單調遞增且連續
